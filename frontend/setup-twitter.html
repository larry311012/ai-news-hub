First, you need a Twitter Developer account to access the Twitter API. This is free and takes about 3 minutes.

Important: You'll need your Twitter/X account credentials. If you don't have a Twitter account yet, create one first at twitter.com/signup

  1. Visit the Developer Portal: Open Developer Portal
  2. Sign in with your Twitter account credentials (the same one you use to tweet)
  3. Complete the developer signup if this is your first time
    • Select "Hobbyist" or "Making a bot" as your use case
    • Describe: "Creating an automated social media posting tool"
    • Accept the Developer Agreement and Terms of Service

Now we'll create a new project and app in the Twitter Developer Portal.

  1. In the Developer Portal, click "Create Project"
  2. Fill in the project details:
    • Project name: "AI News Hub" (or any name you like)
    • Use case: Select "Making a bot"
    • Description: "Automated social media content posting"
  3. Create an App within the project
    • App name: "AI-News-Hub-Bot" (must be unique across all Twitter)
    • If the name is taken, try adding your username or a number

Note: Keep the Twitter Developer Portal tab open. You'll need it in the next steps to get your API keys.

Time to generate your API credentials. These are like passwords that allow our app to post on your behalf.

  1. In your app's settings, find "Keys and tokens"
  2. Click "Regenerate" under "Consumer Keys"
    • You'll see an API Key (about 25 characters)
    • And an API Secret (about 50 characters)
  3. IMPORTANT: Copy both keys immediately!
    • Save them in a secure note or password manager
    • You won't be able to see the API Secret again after closing this window
    • You'll need these in Step 5

Keep Your Keys Secret!

Never share your API keys publicly or commit them to GitHub. Anyone with these keys can post to Twitter as you!

Now we need to configure your app's permissions and callback URL so it can post tweets and complete the OAuth flow.

Part A: Set App Permissions

  1. In your app's settings, click on "App permissions"
  2. Click "Edit" and select "Read and write"
  3. Click "Save"

Part B: Add Callback URL

Troubleshooting: Can't find these settings?

Can't find "App permissions"?

  • Look for "Settings" or "User authentication settings"
  • Try going to Projects & Apps > Your App > Settings

Can't enable OAuth 1.0a?

  • Make sure you're in "Authentication settings"
  • OAuth 1.0a should be under "User authentication set up"
